# **CurioCity 要件定義書**

## 1. プロジェクト概要

* **プロジェクト名**：CurioCity
* **概要**：
  ユーザーが幻想的な架空都市や実在都市を旅しながら、雑学を学べるWebアプリ。
  最初に降り立つ地点はランダムに決定され、地点ごとに雑学と雰囲気に合った背景が表示される。
  芸術作品のようなUIを目指し、毎回の訪問で異なる体験を提供する。

---

## 2. 技術スタック

* **フロントエンド**：Next.js + TypeScript
* **ホスティング**：Firebase Hosting
* **データストア**：静的JSON（事前登録、AI生成データ含む）
* **画像管理**：事前保存（写真素材＋AI生成背景）
* **外部API**（任意利用検討）：

  * Unsplash API / Pexels API（フリー画像取得用）
  * 将来的にStable DiffusionやDALL·Eで背景生成

---

## 3. コンテンツ仕様

* 初期雑学データ数：**1000件**
* 構成：

  * 短文（1〜2行）
  * 詳細解説（約300文字）
  * 雰囲気タグ（Emotion / Setting / Palette）
  * 必要に応じて位置情報（緯度経度）
* 架空都市も登場可能（画像はAI生成で事前準備）
* 背景画像はタグと雑学内容の雰囲気を一致させて選択

## 5. UI/UX仕様

* 最初に降り立つ地点はランダム
* ボタンの位置やデザインはページごとに変化（芸術的UI）
* 背景画像＋雑学テキストのオーバーレイ表示
* ユーザー選択肢あり（単なる「次へ」ではなく分岐）
* 地図表示＋ピン立て演出（地点移動時）
* 雑学ジャンルでのフィルタはなし
* お気に入り機能は実装しない

---

## 6. SNS共有仕様

* テキストリンク＋画像カード生成の両方
* 共有時は短縮版テキストを使用

---

## 7. 非機能要件

* デバイス対応：PC／スマホ両対応（初期はPC最適化）
* 初期ロード時間：5秒以内
* Firebase Hostingのみ利用（Functions等は不要）
* 広告：将来的に検討するが幻想的な世界観を優先

---

## 8. 今後の拡張余地

* 背景画像の自動生成（AI利用）
* ユーザーが雑学を投稿できるモード
* 雰囲気別の旅ルート作成
* 音楽BGM連動（雰囲気タグに応じて変化）

---

## 9. 開発コマンド

* 開発サーバー起動: `npm run dev`
* ビルド: `npm run build`
* リント: `npm run lint`
* デプロイ: `npm run deploy`

### チケット管理システム
- **TICKETS.md** - 全チケットの進捗管理ダッシュボード
- 各チケットファイル内のタスクは `- [ ]` で未完了、`- [x]` で完了を表示
- 成果物セクションの `- [x] 完了時にチェック` で全体の進捗を確認
- 開発時は対象チケットを開いて進捗を更新すること

# チケット #10: 画像表示システムの実装

## 概要
高速起動SimpleAppに段階的な背景画像表示機能を追加し、ユーザーに気づかれないシームレスな画像読み込みを実現

## 現在の課題
- ✅ 即座起動は実現済み（0.1秒未満）
- ❌ 背景画像が表示されない（グラデーションのみ）
- ❌ 雑学ごとの視覚的差別化不足

## 要件

### 段階的画像表示戦略
1. **即座表示**: グラデーション背景で瞬間起動
2. **段階1 (500ms後)**: 低品質プレースホルダー画像表示
3. **段階2 (2秒後)**: 高品質背景画像に差し替え
4. **段階3 (読書中)**: 次の雑学用画像を先読み

## 技術仕様

### 画像URL生成アルゴリズム
```typescript
interface ImageConfig {
  baseUrl: string; // Picsum Photos
  width: number;   // デバイス幅に応じて調整
  height: number;  // 16:9 比率維持
  seed: string;    // 雑学タグのハッシュ
  blur?: number;   // プレースホルダー用ぼかし
  quality?: number; // 段階的品質向上
}
```

### 段階的ロード戦略
```typescript
enum LoadingStage {
  GRADIENT = 'gradient',      // CSS背景のみ
  PLACEHOLDER = 'placeholder', // 低品質ぼかし画像
  STANDARD = 'standard',      // 標準品質画像
  HIGH_QUALITY = 'high'       // 高品質画像
}
```

### パフォーマンス目標
- **段階1**: 500ms以内にプレースホルダー表示
- **段階2**: 2秒以内に標準品質画像表示
- **段階3**: 5秒以内に高品質画像表示
- **キャッシュ命中率**: 80%以上
